Bastionado completo de Ubuntu 24.04 LTS

Kernel PAM Auditd USBGuard Lynis

Este proyecto documenta el proceso completo de bastionado de un sistema Ubuntu 24.04 LTS, siguiendo estándares CIS, NIST y OWASP. Se aplicaron medidas defensivas en múltiples capas: kernel, autenticación, servicios, red, logging y control físico, sin comprometer la funcionalidad del sistema.

Se utilizó Lynis como herramienta de auditoría continua, logrando una mejora de seguridad de 68 a 87 puntos. Cada fase está documentada con comandos, configuraciones y validaciones técnicas.

Firewall dinámico con GeoIP

iptables GeoIP Bash SSH Auth.log

Este proyecto implementa un sistema defensivo automatizado que analiza los intentos fallidos de acceso SSH y bloquea IPs sospechosas según su país de origen.

Utiliza GeoIP para identificar conexiones no españolas y aplica reglas dinámicas con iptables. El script se ejecuta localmente, registra cada acción y permite visualizar los bloqueos en tiempo real.

El resultado es una capa de protección reactiva, ligera y documentable, alineada con buenas prácticas de ciberseguridad operativa.

H-SHIELD — Responsabilidad defensiva en entorno PYME

Ubuntu Server Hardening Observabilidad Detección Respuesta a incidentes

Proyecto central del portfolio orientado a simular la responsabilidad real de ciberseguridad en una PYME. El foco no está en desplegar herramientas, sino en diseñar decisiones defensivas: reducir superficie de ataque, establecer un baseline seguro, detectar desviaciones relevantes y responder a incidentes con criterio.

Incluye hardening de sistemas Linux, monitorización y detección basada en anomalías, análisis de eventos, respuesta proporcional y gestión del riesgo humano en entornos operativos.

Este proyecto está diseñado para ser defendible en entrevista técnica y funcionar como sustituto de experiencia junior, priorizando impacto real frente a complejidad innecesaria.

Nmap Auto Scanner

Python Nmap YAML Cron JSON/CSV

Nmap Auto Scanner automatiza escaneos de red en entornos Linux, genera reportes en JSON y CSV, y compara resultados entre ejecuciones para detectar cambios, nuevos servicios o posibles amenazas. Ideal para auditorías periódicas, bastionado y tareas de blue team.

Desarrollado en Python con integración de python-nmap, utiliza un archivo config.yaml para configurar objetivos, opciones y directorios de salida, además de soportar ejecución programada mediante cron.

Python TTS — Cloned AI Voice

Python ElevenLabs API OpenAI TTS dotenv CLI / Logging

Proyecto de text-to-speech (TTS) desarrollado en Python que convierte texto en voz usando tu voz clonada mediante ElevenLabs o voz neural realista de OpenAI. Diseñado como script CLI, permite integrar síntesis de voz en flujos de trabajo, automatizaciones o contenido multimedia.

Soporta entrada por argumento, archivo o modo interactivo, configuración mediante .env, y salida de audio en MP3/WAV. Incluye sistema de logs y arquitectura extensible para añadir nuevos proveedores como Play.ht o VITS. Licenciado bajo MIT.

Gmail Auto-Responder

Python Gmail API OpenAI API Systemd OAuth 2.0

Proyecto de automatización de respuestas en Gmail mediante Python y la Gmail API, diseñado para ejecutarse de forma autónoma como servicio systemd. Permite responder correos entrantes de manera automática, con opción de respuestas inteligentes generadas por IA (OpenAI).

Integra autenticación OAuth 2.0, gestión de logs detallada y control de duplicados para evitar bucles de respuesta. Su configuración se realiza mediante .env, con intervalos de verificación ajustables. Ideal para tareas de productividad, administración IT o flujos automatizados.

~/criterio-tecnico.md

Este portfolio está diseñado para reflejar criterio técnico aplicado, no para enumerar herramientas ni maximizar cobertura superficial.

  • Arquitectura y decisiones: análisis de elecciones de diseño, compromisos asumidos y exclusiones conscientes bajo restricciones reales.
  • Detección y respuesta: identificación de señales relevantes, gestión del ruido y priorización operativa.
  • Factor humano: evaluación de errores operativos, abuso de privilegios y riesgo organizativo sin enfoques punitivos.
  • Gestión de incidentes: contención, evaluación de impacto real y aprendizaje post-incidente.
  • Exposición y riesgo digital: análisis de cómo la visibilidad pública y profesional amplifica el riesgo humano y operativo, y cómo abordarlo desde un enfoque preventivo y no sensacionalista.

El objetivo no es demostrar exhaustividad, sino criterio defendible bajo restricciones reales, priorizando impacto, riesgo y sostenibilidad frente a complejidad innecesaria.
